# レンダリング方式の選定結果

## 前提条件（2025年3月時点）

| 項目 | 内容 |
|------|------|
| アプリ形態 | React アプリ上で AR を動作させる |
| パフォーマンス目標 | 30fps 程度で可（60fps 必須ではない） |
| 対象デバイス | 数世代前の古い端末も視野に入れる |
| チームスキル | React まで。Three.js / A-Frame / WebGL 経験なし |
| 開発スタイル | AI を活用しながら開発 |
| 将来的な拡張 | 複雑な 3D 配置、複数マーカー、1 マーカーあたり複数コンテンツの可能性 |
| 方針変更 | レンダリング方式を後から変える余裕はない |

---

## 結論: **Three.js を採用**

最初から **AR.js + Three.js** で進めることを推奨する。

---

## 選定理由

### 1. React との統合

| 方式 | React との相性 |
|------|----------------|
| **Three.js** | `@ar-js-org/ar.js-threejs` を npm で導入し、`useRef` / `useEffect` で React コンポーネントに組み込める。状態管理と連携しやすい。`@artcom/react-three-arjs` を使えば React コンポーネントとして記述可能。 |
| A-Frame | iframe や `useRef` で DOM に埋め込む形になり、React のライフサイクルと分離されがち。A-Frame の世界と React の状態を同期する処理が増える。 |

React アプリ内で AR を動かす前提なら、Three.js の方が React と一体化しやすい。

### 2. 将来的な拡張性

- **複雑な 3D 配置**: Three.js はシーングラフを直接操作でき、柔軟に配置可能
- **複数マーカー**: マーカーごとに Group を分けて管理しやすい
- **1 マーカーあたり複数コンテンツ**: 画像・動画・3D を同じマーカーにまとめて配置しやすい

A-Frame でも可能だが、複雑になるほどコンポーネント拡張や Three.js オブジェクトの直接操作に頼ることになり、結果的に Three.js を触ることになる。

### 3. 古い端末・パフォーマンス

- 30fps 目標なら、両方とも達成可能
- 古い端末では、描画コール削減や LOD など細かい最適化が必要になる可能性がある
- Three.js はレンダリングループやメッシュ管理を直接制御できるため、最適化の余地が大きい

### 4. 方針変更不可という制約

- 後から A-Frame → Three.js への移行は工数が大きい
- 将来の拡張を考えると、最初から Three.js で始める方がリスクが小さい

### 5. AI を活用した開発

- 両方ともサンプルはあるが、Three.js + React の組み合わせは情報が多く、AI によるコード生成に適している
- `@ar-js-org/ar.js-threejs` は TypeScript 対応で、型情報が揃っており AI が扱いやすい

---

## 採用する技術スタック

```
AR.js (マーカートラッキング / イメージトラッキング)
  └── @ar-js-org/ar.js-threejs  (npm パッケージ、TypeScript)
        └── Three.js (レンダリング)
              └── React コンポーネントでラップ
```

**オプション**: `@artcom/react-three-arjs` を使うと、React Three Fiber 経由でより React らしい記述が可能。ただし react-three-fiber の習得が必要。

---

## 学習コストへの対応

- Three.js 単体より、**AR.js + Three.js の組み合わせ**に集中する
- `@ar-js-org/ar.js-threejs` の example-vite をベースに、段階的に機能を追加する
- マーカー検出 → 画像表示 → 動画表示 の順で実装し、必要な部分だけ Three.js API を学ぶ
- AI に「AR.js Three.js でマーカーに画像を表示するコード」のように具体的に依頼する

---

## 次のアクション

1. `@ar-js-org/ar.js-threejs` をインストールし、example-vite を参考に最小構成を動かす
2. React コンポーネント（`useRef` + `useEffect`）で AR シーンをマウントする形で統合する
3. マーカー検出 → 画像表示までを実装し、動作確認する
4. 必要に応じて `@artcom/react-three-arjs` の利用を検討する
