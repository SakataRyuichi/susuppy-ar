# 比較と推奨

## 本プロジェクトの前提条件

**対象デバイス**: iOS / Android  
**動作保証**: 各プラットフォームの**デフォルトブラウザ**で動作すること

**AR機能要件**:
- 特定のマーカーを参照し、マーカーを検出したら画像または動画を表示する
- 詳細は [05-ar-requirements.md](./05-ar-requirements.md) を参照

| プラットフォーム | デフォルトブラウザ | WebXR対応 |
|------------------|-------------------|-----------|
| iOS | Safari | ✗ 非対応 |
| Android（多くの端末） | Chrome | ◎ 対応 |
| Android（Samsung） | Samsung Internet | ◎ 対応 |

**結論**: iOS SafariはWebXR未対応のため、**WebXRのみに依存する@react-three/xrは採用不可**。デフォルトブラウザ保証を満たすには、WebXR非依存のソリューションが必要です。（Safari WebXR 検証: 2025年3月2日、[詳細](./02-webxr-and-libraries.md#safari-webxr対応状況の検証記録)）

## プロジェクトタイプ別推奨

### 1. 3DモデルをARで表示（iOS/Android両対応・無料）
**推奨: model-viewer**
- iOS Safari → Quick Look（ネイティブARビューアへ遷移）
- Android Chrome/Samsung Internet → WebXR（ブラウザ内でAR）
- 実装が簡単、完全無料

### 2. ワールドトラッキングAR（iOS Safari内で直接動作）
**推奨: 8th Wall**
- iOS Safari内でワールドトラッキングが可能
- フェイスエフェクト、イメージターゲット等
- 商用利用時は有料（$99/月〜）

### 3. マーカーAR（印刷物連動・完全無料）
**推奨: AR.js**
- iOS Safari、Androidデフォルトブラウザの両方で動作
- マーカー画像をスキャンして3D表示
- 完全オープンソース

### 4. Androidのみ対象の場合
**推奨: @react-three/xr**
- WebXR対応ブラウザ（Android Chrome等）でのみ使用可能
- iOS Safari非対応のため、本プロジェクトの前提条件を満たさない

## 本プロジェクト（meiji-ar）への推奨

### 前提
- React + Web環境
- pnpm / Tailwind / Biome を使用
- **iOS Safari + Androidデフォルトブラウザでの動作保証**

### 推奨アプローチ（デフォルトブラウザ保証を満たすもの）

| ユースケース | 推奨 | 理由 |
|--------------|------|------|
| **マーカー検出で画像・動画表示** | **AR.js** | 本プロジェクトの要件に最適、無料、両プラットフォーム対応 |
| 3DモデルをARで見せる | model-viewer | 無料、両プラットフォーム対応、実装が容易 |
| ワールドトラッキング・フェイスAR | 8th Wall | iOS Safari内で直接動作、機能が豊富（有料） |
| マーカー/印刷物連動AR | AR.js | 無料、両プラットフォーム対応 |

### 本プロジェクトの第一推奨: AR.js

**要件「特定マーカーを検出し、画像・動画を表示」** を満たすため、**AR.js** を第一推奨とします。

- マーカートラッキング（Hiro、バーコード、カスタムパターン）またはイメージトラッキング（任意画像）に対応
- 2D画像、GIF、2D動画の表示が可能
- iOS Safari + Androidデフォルトブラウザで動作
- 完全無料・オープンソース

### model-viewer の補足（iOS）
Quick Lookは「ブラウザ内AR」ではなく、SafariからネイティブのARビューアへ遷移する形式です。ユーザー体験としては「ARボタンタップ → カメラAR画面が開く」となり、実用上は問題なく利用できます。
