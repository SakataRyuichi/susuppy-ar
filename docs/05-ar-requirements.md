# AR要件定義

## 本プロジェクトのAR要件

### 機能要件

| 要件 | 説明 |
|------|------|
| **マーカー方式** | 特定のマーカーを参照し、カメラで検出する |
| **マーカー検出時** | マーカーを特定したらコンテンツを表示する |
| **表示コンテンツ** | 画像および動画を表示する |

### 制約条件（既存）

- 対象デバイス: iOS / Android
- デフォルトブラウザ（Safari / Chrome / Samsung Internet）で動作すること

---

## 推奨ソリューション: AR.js

上記要件を満たし、デフォルトブラウザで動作するのは **AR.js** です。

### AR.jsが要件を満たす理由

| 要件 | AR.jsの対応 |
|------|-------------|
| 特定マーカーの参照 | ◎ マーカートラッキング / イメージトラッキングの両方に対応 |
| マーカー検出 | ◎ カメラでマーカーを検出し、検出時にコンテンツ表示 |
| 画像表示 | ◎ 2D画像、GIFを表示可能 |
| 動画表示 | ◎ 2D動画（HTML5 video）を表示可能 |
| iOS Safari | ◎ WebXR不要で動作 |
| Android デフォルト | ◎ 動作 |
| コスト | ◎ 完全無料・オープンソース |

### AR.jsの2つのマーカー方式

#### 1. マーカートラッキング（Marker Tracking）
- **マーカー種類**: Hiroマーカー、バーコードマーカー、カスタムパターンマーカー
- **特徴**: 軽量、安定したトラッキング、複数マーカー対応が容易
- **用途**: 印刷物にマーカーを印刷して配布する場合

#### 2. イメージトラッキング（Image Tracking / NFT）
- **マーカー種類**: 任意の画像（写真、ロゴ、イラスト等）
- **特徴**: マーカー用の専用画像を用意する必要がない
- **用途**: 既存のポスター、パッケージ、チラシの画像をそのままマーカーとして使用

### 表示可能なコンテンツ（AR.js公式ドキュメントより）

- 2D画像
- GIFアニメーション
- 2D動画（HTML5 video）
- 3Dモデル（glTF等）

### 実装時の注意点（動画表示）

- 動画テクスチャは `markerFound` / `markerLost` イベントで再生・一時停止を制御する
- デバイスや解像度によってサイズ・位置の調整が必要な場合がある
- `playsinline`、`autoplay` 等の属性でモバイルでの再生を最適化する

## 8th Wallの代替案

**イメージターゲット**機能で同様の体験が可能。ただし商用利用時は有料（$99/月〜）。

- 任意の画像をマーカーとして登録可能
- 画像・動画のオーバーレイ表示に対応
- iOS Safari内で直接動作

予算に余裕があり、より高品質なトラッキングが必要な場合は検討の余地あり。
